Reranking

  • Improve relevance of retrieved results by re-ordering based on query and user preferences

  • Models such as LLMs can be used to conduct reranking on results

  • Weaviate has reranking modules out-of-the-box: Cohere, Transformers, Hybrid Search, Multi-Stage Search etc.

Reranking

Multi-Stage Search

- Uses cross-encoder models to re-rank the list of candidates obtained from the search

- Cross-encoders consider the query and each document as a pair and compute a relevance score for each pair

- You can use pre-trained models depending on the domain (from HuggingFace)

Hybrid Search

- Combine scores of vector and BM25F search

- Uses Reciprocal Rank Fusion (RRF) / Relative Score Fusion to rank all entries

- For each result in each list, calculate its reciprocal rank. This is done by taking 1 divided by the rank of the result.

LLM Reranking

Cross Encoder Strategy

The LLM takes a pair of [query, document]Â and outputs a relevance score. This score is used to reorder the documents. The scoring can be bounded by prompting the LLM to output a score within a specific range, such as 1 to 100.

Multiple Documents Strategy

The LLM is given multiple documents along with the query and asked to rank them. This is done by prompting the LLM to output the ranking in a specific format, such as a dictionary with ranks as keys and document IDs as values.
